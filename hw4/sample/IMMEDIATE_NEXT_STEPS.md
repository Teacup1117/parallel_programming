# Block 3 深度診斷

## 關鍵問題：可能是搜索範圍不夠

檢查點：

1. **Block 3 的實際 nbits 值是什麼？**
   - 需要打印出 Block 3 的 nbits 的十六進制和二進制形式

2. **計算出的 target 值是什麼？**
   - target 可能為全 0 或全 1，導致沒有有效的 nonce

3. **搜索的 batch_size 是否足夠？**
   - 如果 nonce 值很大（接近 0xffffffff），100M 的 batch 可能不夠

4. **GPU kernel 的邏輯是否正確？**
   - 可能搜索邏輯在某些邊界情況下失效

---

## 立即建議：使用完全不同的方法

不要依賴 GPU 內存狀態推測，直接查看備份版本是否能通過 Case01：

1. 編譯備份版本 `hw4_backup.cu`
2. 運行在 case01 上
3. 比較輸出

如果備份版本也輸出 `00000000`，說明問題不在你的修改，而在原始代碼邏輯。

如果備份版本通過，那說明你的修改引入了 bug。
